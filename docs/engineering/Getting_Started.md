# Getting Started

**DGTC Platform – Utvecklarguide**

**Status:** Draft v0.1

---

# Syfte

Den här guiden beskriver hur vi utvecklar DGTC Platform tillsammans.

Målet är att göra utvecklingen enkel, konsekvent och begriplig – även för någon som aldrig tidigare har arbetat med GitHub, Flutter eller Visual Studio Code.

Den här guiden är därför skriven för människor, inte för datorer.

---

# Grundprincip

Vi bygger **inte** så snabbt som möjligt.

Vi bygger så bra som möjligt.

Kvalitet går före hastighet.

Förståelse går före komplexitet.

---

# Våra verktyg

Vi använder följande verktyg:

* GitHub – projektets gemensamma arkiv
* GitHub Desktop – för att spara och synkronisera ändringar
* Visual Studio Code – där vi arbetar med dokument och kod
* Flutter – ramverket för mobilappen

---

# Hur vi arbetar

Varje ny idé följer samma process:

1. Idé
2. Diskussion
3. Dokumentation
4. Godkännande
5. Implementation
6. Test
7. Release

Vi hoppar aldrig över dokumentationen för större beslut.

---

# GitHub – förklarat enkelt

Tänk på GitHub som ett digitalt arkivskåp.

Allt som hör till DGTC sparas där:

* dokument
* kod
* bilder
* design
* beslut

GitHub är projektets gemensamma minne.

---

# GitHub Desktop – förklarat enkelt

GitHub Desktop är lastbilen som transporterar filer mellan din dator och GitHub.

Du behöver i början bara känna till tre knappar:

* **Commit** = Spara dina ändringar.
* **Push** = Skicka ändringarna till GitHub.
* **Pull** = Hämta de senaste ändringarna från GitHub.

Det räcker.

---

# Visual Studio Code

Visual Studio Code är vårt arbetsbord.

Här öppnar, läser och ändrar vi filer.

Du behöver inte kunna alla funktioner. Vi lär oss det vi behöver, när vi behöver det.

---

# Vår arbetsmodell

Vi arbetar i små steg.

För varje steg gäller:

1. Vi diskuterar.
2. Dokumentet skrivs.
3. Dokumentet granskas.
4. Dokumentet godkänns.
5. Först därefter börjar implementationen.

---

# Definition of Done

En uppgift är färdig först när:

* den fungerar,
* den är testad,
* dokumentationen är uppdaterad,
* den följer DGTC Way,
* den hjälper användaren att utvecklas.

---

# Om något känns svårt

Om en instruktion känns för teknisk eller svår att förstå ska vi inte gissa.

Vi stannar upp, förenklar och dokumenterar på ett bättre sätt.

Tekniken ska anpassas till människan – inte tvärtom.

---

# Kom ihåg

Du behöver inte kunna allt.

Du behöver bara veta nästa steg.

Det är därför den här guiden finns.
