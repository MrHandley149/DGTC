# GitHub Guide

**DGTC Platform – GitHub Guide**

**Status:** Approved v1.0

---

# Syfte

Den här guiden beskriver hur vi använder GitHub och GitHub Desktop i DGTC-projektet.

Du behöver inte kunna Git.

Du behöver bara förstå vårt arbetsflöde.

---

# Vad är GitHub?

Tänk på GitHub som projektets digitala arkivskåp.

Allt vi bygger sparas där:

* dokument
* kod
* bilder
* design
* beslut

GitHub gör att ingenting försvinner och att vi alltid kan gå tillbaka till tidigare versioner.

---

# Vad är GitHub Desktop?

GitHub Desktop är programmet som kopplar ihop din dator med GitHub.

Vi använder det för att:

* spara ändringar
* skicka dem till GitHub
* hämta nya ändringar

---

# Tre knappar du behöver kunna

## Commit

Commit betyder:

> "Jag vill spara mina ändringar."

En commit är som att ta en ögonblicksbild av projektet.

---

## Push

Push betyder:

> "Skicka mina sparade ändringar till GitHub."

Först efter en Push finns ändringarna även på internet.

---

## Pull

Pull betyder:

> "Hämta de senaste ändringarna från GitHub."

Om flera personer arbetar i projektet börjar man alltid med en Pull.

---

# Vårt arbetsflöde

När vi skapar ett nytt dokument följer vi alltid samma rutin.

### 1. Skapa eller ändra filen

Exempel:

```
docs/foundation/Problem_Statement.md
```

---

### 2. Öppna GitHub Desktop

Programmet visar automatiskt vilka filer som har ändrats.

---

### 3. Skriv ett kort commit-meddelande

Bra exempel:

* Add Learning Model
* Update DGTC Compass
* Create Architecture Overview

Undvik meddelanden som:

* Test
* Ändring
* Fix

Commit-meddelandet ska beskriva vad som faktiskt gjordes.

---

### 4. Klicka på **Commit to main**

Nu är ändringarna sparade på din dator.

---

### 5. Klicka på **Push origin**

Nu skickas ändringarna till GitHub.

Projektet är uppdaterat.

---

# Hur ofta ska jag committa?

Hellre många små commits än få stora.

En bra tumregel är:

> En idé = en commit.

---

# Om något blir fel

GitHub gör att vi nästan alltid kan gå tillbaka till en tidigare version.

Var därför inte rädd för att göra misstag.

Misstag är en naturlig del av utvecklingen.

---

# Våra regler

Vi committar bara sådant som fungerar eller är godkänt.

Vi dokumenterar större beslut innan vi börjar implementera dem.

Vi försöker hålla commit-meddelanden korta och tydliga.

---

# Ordlista

**Repository**
Projektets samlade filer.

**Commit**
Spara en version av projektet.

**Push**
Skicka ändringarna till GitHub.

**Pull**
Hämta de senaste ändringarna från GitHub.

**Branch**
En separat arbetsgren. Vi kommer att börja använda detta längre fram i projektet.

---

# Sammanfattning

För att arbeta med DGTC behöver du i början bara komma ihåg tre steg:

1. Gör ändringen.
2. Commit.
3. Push.

Resten lär vi oss när vi behöver det.
